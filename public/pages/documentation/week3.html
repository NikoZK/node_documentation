Loop Methods

In JavaScript we often use loop methods, these are the most common ones:
[1,2,3].forEach(n => console.log(n))

.forEach() - Runs a function once for every element in an array.
[1,2,3].map(n => n * 2) // [2,4,6]

.map() - Creates a new array by transforming each element.
[1,2,3,4].filter(n => n > 2) // [3,4]

.filter() - Creates a new array with only elements that pass a condition.
[1,2,3].reduce((sum, n) => sum + n, 0) // 6

.reduce() - Combines all values into one single value.
[3,1,2].sort() // [1,2,3]

.sort() - Sorts the array.
[3,1,2].sort((a,b) => a - b)

.find() - Returns the first element that matches a condition.
[1,2,3,4].find(n => n > 2) // 3

.indexOf() - Returns the position (index) of a value in the array.
["a","b","c"].indexOf("b") // 1

Basic rules to remember:

Rule 1: Use loop methods whenever possible
Rule 2: Only use for loops in JavaScript for finger counting
Rule 3: Use map over forEach if you need the data aftwerwards

Why we avoid for loops in JavaScript:

const numbers = [6, 7, 8, 9]

Argument 1: Brevity
Argument 2: Clarity (loop methods communicate intent)
Argument 3: No accidental indexing errors (out of bounds)

for (let i = 0; i < numbers.length; i++) {
    console.log(numbers[i])
}

numbers.forEach((number) => console.log(number))

Argument 4: Avoids for of/in numbers confusion
Argument 5: Able to access both element and index at the same time

let list = [4, 5, 6]

for (let i in list) {
   console.log(i) // "0", "1", "2" <- strings
}

for (let i of list) {
   console.log(i) // 4, 5, 6 <- numbers
}

Argument 6: Avoids side effects
Argument 7: Shorter and more readable/maintainable

let oddIndexSideEffect = []
let index = 0
for (number of numbers) {
    if (index % 2 === 1) {
        oddIndexSideEffect.push(Number(number))
    }
    index++
}
console.log(oddIndexSideEffect)

const oddNumbers = numbers.filter((element, index) => index % 2 === 1)
console.log(oddNumbers)

Argument 8: Doesn't mutate the original list

for (index in numbers) {
    numbers[index] = numbers[index] * 2  
}

Use .map to avoid side effect/mutating the original list

const doubledList = numbers.map((number) => number * 2)

console.log(numbers)
console.log(doubledList)

Anatomy of an URL

An URL(Uniform Resource Locator) is a web address which directs to a specific online resource. Lets take a google search as an example:
https://www.google.com/search?hl=en&q=cool%20search
1[https] 2[://] 3[www] 4[.google] 5[.com] 6[/search] 7[?hl=en&q=example%20search]
  • 1 - Protocol
  • 2 - Protocol identifier
  • 3 - Subdomain
  • 4 - Domain name
  • 5 - Top level domain
  • 6 - Path
  • 7 - Query

Nodemon

Nodemon is a development tool that automatically restarts a Node.js server whenever files change. This removes the need to manually restart the server after every change. It should only be used for development though.

Normally we start a server using: node app.js
With Nodemon we instead run: nodemon app.js or npx nodemon app.js if it's installed as a devDependency, which we talked about last week.

Nodemon watches the project files and restarts the server automatically when changes are detected, like a hot reload.
It optimizes our coding efficiency very much by speeding up the time it takes to restart the server to see changes.

We can also configure our nodemon setup, so the hot reload can happen whenever we want, by changing/adding to the ext(extension) in the nodemon.json file: 

{
    "ext": "js, json, html, css"
}

POST Requests

POST requests are used to send data to the server, usually when creating new resources.
In Express we must enable JSON body parsing before we can access request data: express.json() 

const express = require("express")
const app = express()

app.use(express.json()) // add to express setup

Now we can receive data from a POST request, a simple post request in Express could look like this: 

app.post('/dinosaurs', (req, res) => {
res.send(req.body)  
})

The request can now be accessed through req.body.

Cross Site Scripting (XSS)

Cross Site Scripting (XSS) is a security vulnerability where malicious JavaScript is injected into a webpage.
For example if a user submits input containing script tags:
<script>alert("Hacked!")</script>
If the server returns this directly into the webpage it could execute in the browser.

Frontend prevention

Avoid using innerHTML with untrusted user input. Instead use safer alternatives like textContent. 

element.textContent = userInput

Backend prevention

User input should be sanitized before being stored or returned by the server.
Frameworks and libraries can help sanitize or escape dangerous characters.
We can also manually replace all tags and functional characters for their string counterpart entities like this: 

const containerDiv = document.getElementById('container');
containerDiv.innerHTML = sanitizeXSS(`<img src='x' onerror="alert('XSS')">`);
function sanitizeXSS(string) {
    return string
        .replaceAll('&', '&')
        .replaceAll('<', '<')
        .replaceAll('>', '>')
        .replaceAll('"', '"')
        .replaceAll("'", ''')
        .replaceAll('`', '`')
        .replaceAll('/', '/');
}
