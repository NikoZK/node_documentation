CRUD Rest API, Fetch and Date

CRUD API with Express

A CRUD API supports the four basic operations:
Create, Read, Update and Delete.
In REST APIs we represent these operations with HTTP methods.

  • GET - Read resources
  • POST - Create resources
  • PUT/PATCH - Update resources
  • DELETE - Delete resources
const express = require("express")
const app = express()

app.use(express.json())

const movies = [
    { id: 1, title: "Garlic is as good as ten mothers" },
    { id: 2, title: "Minecraft the Movie" }
]

let nextId = 3

We have 2 movies and a let nextId to simulate and comply with auto incrementation. 

app.get("/movies", (req, res) => {
    res.send({ data: movies })
})

app.get("/movies/:id", (req, res) => {
    const movieId = Number(req.params.id)
    const foundMovie = movies.find(movie => movie.id === movieId)

    if (!foundMovie) {
        return res.status(404).send({ errorMessage: "Movie not found" })
    }

    res.send({ data: foundMovie })
})

GET method for movies with error handling. 

app.post("/movies", (req, res) => {
    const newMovie = {
        id: nextId++,
        title: req.body.title
    }

    movies.push(newMovie)
    res.status(201).send({ data: newMovie })
})

POST method with HTTP status code and auto incrementation. 

app.patch("/movies/:id", (req, res) => {
    const movieId = Number(req.params.id)
    const foundMovieIndex = movies.findIndex(movie => movie.id === movieId)

    if (foundMovieIndex === -1) {
        return res.status(404).send({ errorMessage: "Movie not found" })
    }

    const updatedMovie = { ...movies[foundMovieIndex], ...req.body }
    movies[foundMovieIndex] = updatedMovie

    res.send({ data: updatedMovie })
})

PATCH method with error handling. 

app.delete("/movies/:id", (req, res) => {
    const movieId = Number(req.params.id)
    const foundMovieIndex = movies.findIndex(movie => movie.id === movieId)

    if (foundMovieIndex === -1) {
        return res.status(404).send({ errorMessage: "Movie not found" })
    }

    movies.splice(foundMovieIndex, 1)
    res.send({ message: "Movie deleted" })
})

DELETE method which checks if movie even exists. 

app.listen(8080, () => {
    console.log("Server is running on port 8080")
})

This is an example of full CRUD for the resource "movies", which we made in week 3.

Fetch Requests

We can use fetch() in the browser to communicate with our backend.
Below is an example of fetching movies from our Express server and displaying them on the page. 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Movies</title>
</head>
<body>
    <h1>Movies</h1>

    <h3 id="movie-output"></h3>

    <script>
        fetch('/movies')
        .then((response) => response.json())
        .then((result) => {
            const output = document.getElementById('movie-output')

            output.textContent = JSON.stringify(result.data)
        });
    </script>

</body>
</html>

The request fetches data from the server and displays it on the page through the "movie-output" id.

Date

JavaScript has a built-in Date object which can be used to get the current date and time. 

console.log(new Date()) // Current date and time
console.log(Date()) // Current local date as a string
console.log(Date.now()) // Milliseconds since January 1st 1970

We can also extract specific parts of the date, by manually mapping the value to an index of an array, which corresponds to the month or day: 

const months = ["January","February","March","April","May","June","July","August","September","October","November","December"]
const days = ["Sunday","Monday","Tuesday","Wednesday","Thursday","Friday","Saturday"]

const currentMonth = new Date().getMonth()
const currentDay = new Date().getDay()

console.log(months[currentMonth])
console.log(days[currentDay])

Or we can use toLocaleString() to automatically extract the date and time : 

console.log(new Date().toLocaleString("default", { month: "long" }))
console.log(new Date().toLocaleString("default", { weekday: "long" }))
