Imports, Static files and redirection

Serving Static Files in Express

In Express we can serve static files such as HTML, CSS, JavaScript and images by using:
app.use(express.static("public"))
This means files inside the public folder can be accessed directly by the browser.

This is important for security, because we only expose the files we need to show in the frontend and not the whole project folder.

Import and Export in Frontend JavaScript

"ES Modules (ESM) are the official, standardized format for packaging JavaScript code for reuse, featuring built-in import and export statements"

In the frontend we can use ES Modules by adding the type="module" attribute to the script tag. 

<script type="module" src="./cookies.js"></script>

We also need to add it to our package.json: 

{
  "type": "module",
  "dependencies": {
    "express": "^5.2.1"
  }
}

This allows us to use import and export in browser JavaScript files.
This file below is cookieFactory.js: 

function getNewCookieBatch() {
    return ['🍪', '🥛', '🍪', '🍪']
}

export default {
    getNewCookieBatch
}

And this file below is cookies.js and it imports cookieFactory.js: 

import cookieFactory from './cookieFactory.js'

console.log(cookieFactory.getNewCookieBatch())

We export a default object from one file and import it in another file.

CommonJS vs ES Modules

There are two module systems which Node.js supports: CommonJS and ES Modules.
But CommonJS is the classic but outdated way of doing it, and almost all modern applications use ES Modules.

CommonJS

const cookie = require('./cookie.json')

function getCookie() {
    return cookie
}

module.exports = {
    getCookie
}

ES Modules

import express from 'express'

export function esModuleCookieFactory() {
    return "On a break..."
}

CommonJS uses require() and module.exports, while ES Modules simply uses import and export.

Redirection

Redirection means sending the user from one URL to another. There are multiple ways to do this.

Client-side redirection

Client-side redirection happens in the browser.
I will show three examples:

  • Anchor tags
  • Meta refresh
  • JavaScript with location.href or location.replace()

Anchor tag which redirects to "/" instantly: 

<a href="/">Take me awayyyyyy</a>

We can also use a meta tag to redirect: 

 <meta http-equiv="refresh" content="3;/" />

Or with JavaScript functionality: 

setTimeout(() => {
    location.href = '/'
}, 3000)

Server-side redirection

Server-side redirection happens on the server, before the browser receives the final page. In Express this can be done with: 

app.get("/go-home", (req, res) => {
    res.redirect("/")
})

The difference is that client-side redirection is triggered in the browser, while server-side redirection is triggered by the backend.
