Variables and Scoping

JavaScript has 3 ways of defining variables (realistically 2)

const is used for variables which should not be reassigned.

let is used for variables which need to be reassigned later.

var is considered outdated and should generally be avoided. One of the biggest reasons is that it is function-scoped instead of block-scoped.

Block scope means that variables only exist inside the block they were declared in. 

{
    const name = "Nadia"
}

console.log(name) // ReferenceError

With var, variables inside blocks are not isolated and can override each other. 

{
    var someVariable = true;
    {
        var someVariable = false;
    }
    console.log(someVariable);
}

Here someVariable logs false because both var declarations refer to the same variable. 

{
    let someVariable = true;
    {
        let someVariable = false;
    }
    console.log(someVariable);
}

Using let instead logs true because the inner variable is scoped to its own block.

const also prevents reassignment: 

const age = 25
age = 27 // Error

However objects and arrays declared with const can still have their contents modified. 

const user = { name: "Nadia" }

user.name = "Niko" // allowed

Callback Functions

A callback function is a function that is passed as an argument to another function and executed later. 

function processUser(name, callback) {
    callback(name)
}

processUser("Niko", (name) => {
    console.log("Hello " + name)
})

Callbacks are heavily used in JavaScript for asynchronous behavior and event handling.
They can be written in several ways: 

// Named function
function eatAction(name) {
    console.log(name)
}

processUser("Niko", eatAction)

// Anonymous function
processUser("Niko", function(name){
    console.log(name)
})

// Arrow function
processUser("Niko", name => console.log(name))

NPM (Node Package Manager)

NPM is the default package manager for Node.js. It's a language specific package manager, like Maven/ Gradle is Java or Pip for python.

We can also use Yarn or pnpm as alternatives. It is used to install, manage and share JavaScript packages.
Packages are reusable libraries that other developers have published.

npm install express

This command downloads the Express framework and all its dependencies.

package.json

package.json is the configuration file for a Node.js project. It describes the project and manages its dependencies.
It usually contains the information below, but may also just be the dependencies for simple projects.

  • Project name
  • Version
  • Scripts
  • Dependencies
  • Project metadata
{
    "dependencies": {
        "express": "5.2.1"
    }
}

node_modules

The node_modules folder contains all installed dependencies of the project.
After running:

npm install

NPM downloads every required package and stores them inside this folder.

This folder should never be pushed to GitHub because:

  • It can become extremely large
  • Dependencies can always be reinstalled by other dev's by just running npm install
  • The package.json and package-lock.json already define them

Therefore we add it to .gitignore.

Express.js

Express.js is a popular Node.js web application framework for routing and HTTP requests/reponse, due to being minimal and very easy to set up. 

// step 1 - import express
const express = require('express')

// step 2 - instantiate express
const app = express()


// step 3 - have the server listen on port 8080
app.listen(8080)

The server now runs on http://localhost:8080 and can responds to requests.

It can also be optimized a bit, by tweaking the setup a bit:

// we can make the first two parts a one-liner:
const app = require('express')();


// and by defining the port in process.env and using short circuit syntax 
as well as adding some error handling and port + enviroment logging:

const PORT = process.env.PORT || 8080

const server = app.listen(PORT, (error) => {
    if (error) {
        console.log("Error starting the server")
    }
    console.log("Server is running on port", server.address().port, "The enviroment is:", process.env.NODE_ENV)
})

We now have a clean and optimized Express.js server setup.

Client Server Model

The client-server model is the basic structure of the web, explaining what happens when clients and servers interact.

  • Client - The client makes a request from their browser or mobile app
  • Server - The system responds with data or resources

Example:

  • Client sends HTTP request
  • Server processes the request
  • Server sends HTTP response
  • Client receives and displays the data

The most basic example is when visiting a website your browser acts as the client and the web server returns HTML, CSS and JavaScript and then you see the content.

GET Requests

GET requests are used to retrieve data from a server.
Data can be sent through using path parameters or with query parameters.

Path variable: 

app.get('/cars/:carModel/:year', (req, res) => {
    console.log(req.params);
    res.send({ data: `
        Your ${req.params.carModel} is very nice.
        Is it from the year ${req.params.year}?
    `});
});

So when you access http://localhost:8080/cars/mazda+mx5/1998
The response becomes: { data: "Your mazda mx5 is very nice.Is it from the year 1998?" }

Query string: 

app.get('/bag', (req, res) => {
    res.send({ data: req.query });
});

Here we use the URL parameters with req.query.
So when we access the URL: http://localhost:8080/bag?color=red&size=small
The response becomes: { data: { color: "red", size: "small" } }
