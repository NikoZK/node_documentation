Variables and datatypes

Variables

JavaScript has 3 ways of defining a variable(Realistically 2)

const is used for variables which do not want to be reassigned later.

let is used for variables which want to be reassigned later.

var is just not used anymore avoid it.
One of the biggest reasons is, that its function-scoped, not block-scoped. This means that variables declared inside loops, if statements, or other blocks are not confined to that block, but can be accessed outside it, which can lead to bugs. 

                
{
    var someVariable = true;
    {
        var someVariable = false;
    }
    console.log(someVariable);
}

Here someVariable would log "false" because the inner var overrides the outer, because they are practically the same variable. 

                
{
    let someVariable = true;
    {
        let someVariable = false;
    }
    console.log(someVariable);
}

While using let instead it will log "true", because the inner let is not the same variable as the outer.

Datatypes

In JavaScript we have 8 datatypes

  • String - Text value
  • Boolean - True or false
  • Number - Numeric value
  • BigInt - Huge numeric value
  • Object - Key: Value pair
  • Symbol - Immutable and unique symbol value
  • null - Explicitly set to no value
  • undefinded - No value asssigned yet

Type coercion

In JavaScript datatypes can automatically convert from one datatype to another if necessary.


5 + "10" = "510"
"5" + 10 = "510"

"5" - 2 = 3
"5" * 2  = 10
"5" / 2 = 2.5

We can see that when using +, the coercion prefers Strings, thats because + is used for number addition AND String concatenation.
While -, *, / and %only works with numbers, and therefore results in a Number coercion. 

                
Boolean("True") = True
Boolean("False") = True
Boolean("") = False
Boolean(0) = False
Boolean(1) = True

Booleans also do type coercion and will result True to everything EXCEPT falsy values.
These are the only falsy values in JavaScript: 

            
false
0
-0
0n
"" 
null
undefined
NaN

Everything else will equal True.

Strict and loose equality

If we want to avoid type coercion, we can use strict equality( === )
Then JavaScript checks if BOTH the types and values are the same: 

                
10 === 10 = true
10 === "10" = false
true === true = true
true === 1 = false
true === "true" = false
false === false = true

For comparison there is also loose equality ( == ), which does type coercion FIRST and then compares.
Therefore all the examples above, would become true.

String defining

In javaScript we have 3 ways of defining a string
Single qoutes: ''
Double qoutes: ""
Backticks: ``
You cannot use single or double qoutes inside of single or double qoutes, that would end the string. Therefore we can use single qoutes inside of double qoutes or double qoutes inside of single qoutes to avoid that problem. 

                    
// This wont work, since the string ends after the second qoutation mark.
const name = "Niko "The destroyer of worlds" " 

// This works!
const name = "Niko 'The destroyer of worlds' "
const name = 'Niko "The destroyer of worlds" '

Backticks can include both and have interpolations available like: ( ${} ), as well as being multi-line. 

                
const myName = `My name is ${firstName} ${lastName}
and i like carrot juice`

You could use backticks anytime and always, but technically are a bit slower to process due to the interpreter scanning for interpolations and it can be harder to read.

Loop Methods

In JavaScript we often use loop methods, these are the most common ones:
[1,2,3].forEach(n => console.log(n))

.forEach() - Runs a function once for every element in an array.
[1,2,3].map(n => n * 2) // [2,4,6]

.map() - Creates a new array by transforming each element.
[1,2,3,4].filter(n => n > 2) // [3,4]

.filter() - Creates a new array with only elements that pass a condition.
[1,2,3].reduce((sum, n) => sum + n, 0) // 6

.reduce() - Combines all values into one single value.
[3,1,2].sort() // [1,2,3]

.sort() - Sorts the array.
[3,1,2].sort((a,b) => a - b)

.find() - Returns the first element that matches a condition.
[1,2,3,4].find(n => n > 2) // 3

.indexOf() - Returns the position (index) of a value in the array.
["a","b","c"].indexOf("b") // 1

Basic rules to remember:

Rule 1: Use loop methods whenever possible
Rule 2: Only use for loops in JavaScript for finger counting
Rule 3: Use map over forEach if you need the data aftwerwards

Why we avoid for loops in JavaScript:

                
const numbers = [6, 7, 8, 9];

/* 
Argument 1: Brevity
Argument 2: Clarity (loop methods communicate intent)
Argument 3: No accidental indexing errors (out of bounds)
*/
for (let i = 0; i < numbers.length; i++) {
    console.log(numbers[i]);
}

numbers.forEach((number) => console.log(number));

/*
Argument 4: Avoids for of/in numbers confusion
Argument 5: Able to access both element and index at the same time
*/
let list = [4, 5, 6];

for (let i in list) {
   console.log(i); // "0", "1", "2" <- strings
}

for (let i of list) {
   console.log(i); // 4, 5, 6 <- numbers
}

/* 
Argument 6: Avoids side effects
Argument 7: Shorter and more readable/maintainable
*/
let oddIndexSideEffect = [];
let index = 0;
for (number of numbers) {
    if (index % 2 === 1) {
        oddIndexSideEffect.push(Number(number));
    }
    index++;
}
console.log(oddIndexSideEffect);

const oddNumbers = numbers.filter((element, index) => index % 2 === 1);
console.log(oddNumbers);

/* Argument 8: Doesn't mutate the original list */
for (index in numbers) {
    numbers[index] = numbers[index] * 2;    
}

// Use .map to avoid side effect / mutating the original list
const doubledList = numbers.map((number) => number * 2);

console.log(numbers);
console.log(doubledList);

REST API Design

(RE)presentational (S)tate (T)ransfer application programming interface, is an architectural style that's commonly seen as the standard for designing and building the networked applications that power the web.

One of the core principles of REST is using the right HTTP methods for the right action and representing them in the right order.
  • GET - Retrieves data (Read)
  • POST - Creates data (Create)
  • PUT - Replaces data (Update)
  • PATCH - Replaces part of data (Update)
  • DELETE - Deletes data (Delete)

These methods should represent resources and not actions and be plural.

    Name - Method - Description
  • /beers - GET - Return all beer resourcses
  • /beers{id} - GET - Return a beer resource by id
  • /beers - POST - Create a beer resource
  • /beers{id} - PUT - Update a beer resource by id
  • /beers{id} - PATCH - Update a part of a beer resource by id
  • /beers{id} - DELETE - Delete a beer resource by id
